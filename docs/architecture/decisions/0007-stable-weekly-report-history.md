# ADR-0007: 稳定的成长周报历史

**Status:** Accepted
**Date:** 2026-06-19

## Context

如果 pending 任务被物理删除，或周末后的补完成继续参与聚合，过去周报的任务分母、分子和完成率会随当前操作变化，家庭无法解释同一周报为何前后不同。任务实际时长和成长记录时长同时相加还会重复计时。

## Decision

GrowthTask 不做物理删除。pending 删除转为 `cancelled` 并记录 `cancelledAt`，completed/confirmed 删除转为 `archived`。历史周只使用周末截止时可见的状态：周末前取消的任务不进分母，周末后的取消仍保留；周末后的补完成不进分子。时长只汇总 GrowthLog。已结束周首次成功生成后冻结统计快照，反馈字段仍可更新。

## Alternatives

- 每次读取都按当前状态重算：实现简单，但历史结果会漂移。
- pending 继续物理删除并另建审计表：能够追溯，但增加双写和聚合复杂度。
- 同时汇总任务与记录时长：信息更多，但无法可靠判断两者是否为同一次活动。

## Consequences

任务集合会保留 cancelled/archived 行，需要状态索引和保留策略。当前周可以随源数据变化；已结束周的统计不可回写。任务列表默认可隐藏终态，但周报仓储必须读取截止时间字段。任务 actualMinutes 仍用于任务详情，不作为周报总时长来源。

## Validation

测试覆盖周末前后取消、跨周补完成、零任务周、任务与 GrowthLog 双录时长，以及已冻结周报重复读取。
