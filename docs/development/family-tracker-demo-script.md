# 家庭成长追踪 MVP 演示脚本

**适用基线：** Task 11  
**演示范围：** 家长建家、五育任务、孩子完成、家长确认、成长记录、错题与私有图片、周报、提醒、星星奖励及身份隔离

## 1. 准备

1. 在仓库根目录执行 **node tests/e2e/task11/startFamilyApp.js**。
2. 等待终端显示 **Task 11 family app ready**。
3. 浏览器打开 **http://127.0.0.1:3100/register**。
4. 演示完成后在启动终端按 **Ctrl+C**。运行时会关闭前端、网关、六个服务、MongoDB 副本集并删除临时媒体。

不要在截图、聊天或工单中粘贴 bearer token、PIN、签名媒体 URL 或运行时密钥。

## 2. 家长建家

1. 注册家长账号。用户名使用 20 字符以内的唯一值，例如 **demo_parent_01**；密码至少 8 位。
2. 创建家庭“演示成长之家”。
3. 从家长导航进入“孩子”。
4. 记录页面显示的“家庭 ID”。
5. 添加“小明”（3 年级）和“小红”（2 年级）。
6. 记录小明的“孩子 ID”，为小明设置 PIN 2468，为小红设置 PIN 1357。

预期：页面只显示家庭 ID、孩子 ID 和 PIN 更新结果，不回显 PIN。

## 3. 创建五育任务

进入“任务”，为小明创建当天截止的五个任务：

| 维度 | 标题 | 领域 | 类型 |
| --- | --- | --- | --- |
| 德育 | 帮助同学 | 友善 | 习惯 |
| 智育 | 完成数学练习 | 数学 | 练习 |
| 体育 | 跳绳 500 个 | 体能 | 锻炼 |
| 美育 | 练习钢琴 | 音乐 | 练习 |
| 劳育 | 整理房间 | 家务 | 劳动 |

体育任务填写目标数量 500、单位“个”。预期：五项均为“待完成”，切换到小红后列表为空。

## 4. 孩子完成任务

1. 新建浏览器隐私窗口，打开 **http://127.0.0.1:3100/child/login**。
2. 输入家长页显示的家庭 ID、小明的孩子 ID 和 PIN 2468。
3. 打开“跳绳 500 个”，填写实际用时 18、实际数量 500 和一句话“我按计划完成了。”。
4. 提交完成情况。

预期：孩子端显示“等待家长确认”，孩子导航不包含家长功能。

## 5. 家长确认与成长证据

1. 回到家长窗口刷新“任务”，确认“跳绳 500 个”，填写反馈“完成得很认真。”。
2. 进入“记录”，新增当天体育记录“跳绳训练完成”，时长 18 分钟。
3. 进入“错题”，新增数学知识点“两位数乘法”，原因选“计算错误”，正确答案填 84。
4. 上传一张 PNG、JPEG 或 WebP 题目图片，点击“查看题目图片”，确认只通过授权预览显示，再保存错题。

预期：任务为“已确认”；星星流水增加 +1；错题图片没有公开存储路径。

## 6. 周报、提醒与奖励

1. 进入“周报”。确认记录天数为 1、成长投入为 18 分钟、新增错题为 1。
2. 进入“提醒”，启用“周报提醒”，提醒日选“周日”，保存设置。
3. 进入“星星与奖励”，确认当前星星为 1。
4. 新建奖励“选择周末家庭活动”，所需星星 1，执行兑换。

预期：兑换后余额为 0，仅产生一次扣减流水。

## 7. 隔离与失效检查

1. 在孩子窗口直接打开 **/app/today**。预期跳转家长登录且不显示家长导航。
2. 家长在“孩子”页把小明 PIN 重置为 8642。
3. 刷新孩子窗口中的受保护页面。

预期：旧孩子会话被清除，返回孩子登录页并显示会话过期；旧 PIN 不再可用，新 PIN 可登录。兄弟姐妹或其他家庭访问小明数据应返回 403，响应不得包含小明的私有内容。

## 8. 故障处理

- 页面未启动：确认 3100 端口未被占用，停止旧进程后重新执行启动命令。
- 首次启动较慢：等待 CRA 编译和 MongoDB 内存副本集就绪，不要并行启动第二套运行时。
- 图片失败：确认格式为 JPEG、PNG 或 WebP，大小不超过 10 MiB。
- 测试失败：执行 **npm run test:task11**。只在失败时检查 **playwright-report/** 和 **test-results/**，不要把其中的运行时数据提交到 Git。
